# Reviewer Protocol

1. Reviewer receives source sentence.
2. Reviewer builds A1-A6 packet.
3. Resolver receives packet, ideally without source sentence.
4. Resolver assigns Remainder / Orientation / Operator.
5. Compare with expected address.
6. Record agreement, disagreement, and reason.
7. For translations, repeat without relying on the source sentence.
8. Do not treat agreement as proof by itself.

## Notes

- Reviewers should record uncertainty rather than forcing agreement.
- Rival readings should be preserved, not hidden.
- The same sentence may be tested across multiple reviewers and models.
- Translation tests should include native-language review where possible.
